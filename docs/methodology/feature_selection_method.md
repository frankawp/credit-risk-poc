# 特征筛选与去相关方法

## 1. 目标

特征筛选阶段的目标不是“尽量删特征”，而是：

- 删除无效变量
- 删除重复变量
- 保留有效且稳定的代表变量
- 让最终宽表更易解释、更易维护

## 2. 当前项目采用的筛选逻辑

当前筛选流程分四层：

### 2.1 基础过滤

先删掉明显无价值的变量：

- 高缺失
- 常数列
- 近常数列
- 非空样本过少
- 单变量几乎没有信号的变量

### 2.2 单变量评分

每个特征计算：

- `univariate_roc_auc`
- `univariate_pr_auc`
- `recall_at_topk`
- `lift_top_decile`
- `missing_rate`
- `outlier_rate`
- `importance_full_model`

这些指标共同组成 scorecard，而不是只看一个数字。

### 2.3 相关性分组

对候选变量做相关性分析：

- 默认用 `Spearman`
- 当绝对相关性达到阈值时，划入同一相关组
- 每组保留一个主代表

当前项目还做了一点业务约束：

- 如果一个组里同时存在连续强度变量和二值规则变量
- 且两者相关性没有高到极端重合
- 则可以同时保留一强一弱两个代表

### 2.4 增量验证

筛选完成后，不直接假设“删完更好”，而是重新训练：

- 全量特征模型
- 筛选后模型

比较：

- ROC-AUC
- PR-AUC
- Recall@TopK

只有在指标损失可接受的前提下，筛选结果才算有效。

## 3. 为什么要去相关

在变量挖掘里，不少变量是同一母题下的派生特征。例如：

- `org_region_bad_rate` 和 `org_region_risk_rank`
- `recent_application_gap_min` 和 `recent_application_gap_std`
- `atm_cashout_ratio_max` 和 `atm_cashout_ratio_mean`

如果全部保留，会带来：

- 模型解释重复
- 变量手册臃肿
- 后续维护成本变高
- 新特征难以识别真实增益

因此，去相关是变量工程的一部分，而不是模型调参的附属动作。

## 4. 相关组内保留规则

当前项目默认按以下顺序保留代表变量：

1. 单变量效果更强
2. 缺失率更低
3. 分布更稳，极端值更少
4. 业务解释更直接
5. 更适合后续规则化落地

这意味着：

- 并不是 importance 高就一定保留
- 也不是业务好解释就一定保留
- 而是“效果、稳定性、可解释性”综合平衡

## 5. 当前项目中的经验结论

从当前第一阶段结果看，筛选层有几个实际启示：

- 群体风险编码类特征通常要去重，否则容易重复表达同一信号
- 极端稀疏的二值规则更适合保留为辅助规则，而不是全靠模型去学
- 比率类变量要特别注意爆炸值，否则相关性和单变量效果会失真
- 贷后恶化类变量往往互相关较高，需要选出最稳定代表

## 6. 如何在后续新增变量时使用这套方法

新增变量后，不应直接塞进最终特征集，而应按顺序做：

1. 先补到变量手册
2. 再进入 scorecard
3. 再看是否进入相关组
4. 再看最终是否进入 selected feature set

只有经过这四步，变量才算正式进入方法体系。
