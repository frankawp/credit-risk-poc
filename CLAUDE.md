# Credit Risk POC - 信贷变量挖掘技能开发项目

## 工程定位

本工程的核心目标是**产出 Claude Code 可用的技能（Skill）**，而非开发纯 Python 框架。

### 主要交付物

**信贷变量挖掘技能**（`feature-mining-orchestrator`）：一个可独立安装、对业务人员友好的 Claude Code 技能，能够：
- 接收业务人员提供的数据表
- 与用户协作理解业务问题和数据特点
- 动态设计、实现、验证风险变量
- 迭代优化，产出有价值的变量集

### 技术支撑

`src/dual_engine/` 是技能的底层引擎，提供：
- `auto_features`: 基于 Featuretools 的自动化统计挖掘
- `semantic_features`: 业务语义驱动的变量组
- `composite_features`: 有约束的组合特征
- `selection`: 特征筛选流程

引擎代码是手段，技能才是目的。

---

## AI 在变量挖掘中的角色

AI 不是"脚本调度器"，而是"变量挖掘伙伴"。

### 核心能力

1. **数据理解**：主动探索数据结构、分布、业务含义
2. **假设设计**：基于业务知识提出变量假设
3. **代码实现**：动态生成 Python 函数实现变量
4. **效果验证**：评估变量效果，迭代优化
5. **知识沉淀**：记录假设来源、迭代过程、效果证据

### 工作原则

- 先理解数据，再设计变量
- 每个变量都有业务假设支撑
- 效果不好时主动调整策略
- 对用户的业务问题保持敏感

## 开发规范

### 变量设计

- 每个新变量需记录：变量名、计算逻辑、业务假设、预期效果
- 复杂变量需要代码审查
- 效果验证后更新状态（候选/入选/废弃）

### 代码风格

- 中文注释和文档
- 类型注解
- 单元测试覆盖核心逻辑

### 文档更新

- 新增变量时更新 `references/implemented_features.md`
- 方法论变更时更新 `docs/methodology/`
